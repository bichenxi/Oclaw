为 OpenClaw Agent 添加新的站点专属技能文件。

## 参数

使用方式：`/skill-add <站点名称>`
例如：`/skill-add 微博` 或 `/skill-add bilibili`

## 执行步骤

1. **分析目标站点**：读取 `openclaw-skill/SKILL.md` 了解通用能力，明确目标站点的特殊性（特有的 API、交互模式、反爬机制等）

2. **读取现有范例**：读取 `openclaw-skill/xhs.md` 作为格式参考

3. **创建新技能文件** `openclaw-skill/<站点英文名>.md`，包含以下结构：

```
---
name: claw-<站点名>
version: 1.0
description: 通过 Claw Browser 自主操控<站点>的专属技能。
---

# <站点> 自主操控技能

> 本文件是 SKILL.md 的站点专属子技能。通用能力见主技能文档。

## ⚠️ 约束
（该站点的核心限制）

## 🚨 任务完成条件
（明确什么算"完成"，避免 Agent 浅尝辄止）

## 一、专属 API
（该站点专属端点表格）

## 二、典型工作流
（3-5 个典型场景的操作步骤）

## 三、注意事项
（反爬、登录态、弹窗等特殊处理）
```

4. **更新主技能文件**：在 `openclaw-skill/SKILL.md` 的"站点专属技能"表格中加入新条目（站点、技能文件、触发关键词）

5. 输出创建的文件内容摘要
