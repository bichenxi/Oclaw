## 第一步：读取现状

1. 读取 `src-tauri/src/api.rs` —— Rust 端当前暴露的所有 HTTP 端点（路由、参数、返回值）

## 第二步：差异分析

对比代码与技能文档，找出：
- **新增端点**：代码中有但技能文档中没有 → 需要加入
- **变更端点**：参数、返回值、行为有变化 → 需要更新描述
- **删除端点**：代码中已移除但文档中还有 → 需要删除或标注废弃
- **描述不准确**：技能文档的说明与实际行为不符 → 需要修正

## 第三步：更新技能文件

根据差异分析的结果，更新 `openclaw-skill/SKILL.md`（和相关子技能文件）。

**写作规范**（技能文件的受众是 OpenClaw AI Agent，不是人类开发者）：
- 语言：简洁、指令式、以操作为导向
- 表格：用 Markdown 表格列出端点、方法、Body、返回值
- 约束：用 `⚠️` 或 `❌/✅` 标注必须遵守的规则
- 场景：用编号步骤写出典型操作流程
